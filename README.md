# Пакет агента уведомлений о незавершенном обучении

## Описание

Агент автоматически отправляет уведомления сотрудникам о незавершенных курсах и тестах каждый день в 8:00 утра.

## Компоненты пакета

### Файлы

- `notification.agent.ts` - Основной код агента
- `agent_schedule.xml` - Конфигурация планировщика
- `notification_type.xml` - Тип уведомления в системе
- `notification_template.xml` - Шаблон уведомления
- `README.md` - Документация

### Возможности

✅ **Автоматический запуск**: Каждый день в 8:00 утра  
✅ **Фильтрация**: Только неуволенные сотрудники  
✅ **Группировка**: Одно уведомление на сотрудника  
✅ **Защита от дублирования**: Не отправляет повторно в один день  
✅ **Обработка ошибок**: Логирование и восстановление  
✅ **Производительность**: Оптимизированные SQL запросы

## Установка

### 1. Развертывание файлов

```bash
# Скопировать файлы в соответствующие директории WebTutor
cp notification.agent.ts /webtutor/agents/
cp agent_schedule.xml /webtutor/config/schedules/
cp notification_type.xml /webtutor/config/notifications/
cp notification_template.xml /webtutor/config/templates/
```

### 2. Регистрация в системе

1. **Планировщик**: Зарегистрировать агент в планировщике WebTutor
2. **Тип уведомления**: Создать новый тип через админ-панель
3. **Шаблон**: Загрузить шаблон в систему шаблонов

### 3. Настройка базы данных

Убедитесь в наличии необходимых таблиц:

- `active_learnings` (незавершенные курсы)
- `active_testings` (незавершенные тесты)
- `collaborators` (сотрудники)
- `courses` (курсы)
- `tests` (тесты)
- `learning_states` (состояния обучения)
- `testing_states` (состояния тестирования)
- `notifications` (уведомления)

## Мониторинг

### Логи

Агент создает подробные логи:

- `INFO`: Общая информация о работе
- `DEBUG`: Детали отправки уведомлений
- `ERROR`: Ошибки выполнения
- `WARN`: Предупреждения
